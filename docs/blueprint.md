# **App Name**: RegistraTrack

## Core Features:

- Dashboard Overview: Display a real-time summary of registration statistics, including total, processed, pending review, rejected, processing, and failed registrations, with visual indicators.
- Recent Registrations List: Show a table of recent registration submissions with key details like name, date, status, and location.
- User Authentication: Allow secure login and user management for accessing the tracker dashboard.
- Data Storage & Retrieval: Store and efficiently retrieve all registration entries and their associated metadata, such as status and submission details.
- Navigation System: Provide a clear top navigation bar to switch between the dashboard and other key sections like full registrations list and reports (as they are developed).
- AI-Assisted Status Suggestion: A tool that analyzes incoming registration details (e.g., submission content, required fields) and suggests an initial processing status (e.g., 'pending review', 'processing') to streamline manual classification.
- Cross-platform Compatibility: The application will be developed to support mobile (iOS, Android), desktop (Windows, macOS, Linux), and web platforms, ensuring a consistent user experience across all environments, leveraging technologies like Flutter.

## Style Guidelines:

- Primary action and brand color: A deep, professional blue (#1F50A3) reflecting reliability and data integrity. This will be used for interactive elements and prominent branding.
- Background color: A very light, desaturated blue-gray (#ECF1F5) providing a clean, spacious backdrop for the data-rich dashboard elements.
- Accent color: A vibrant, clear cyan (#1FC2F5) used for highlighting important notifications or secondary calls-to-action, drawing attention without being distracting.
- Main font for all text: 'Inter' (sans-serif) for its modern, clean, and highly readable characteristics suitable for data display and a professional aesthetic.
- Use a consistent set of clean, minimalist outline icons for navigation and actions, complemented by distinct solid fill icons (green for success, red for rejected/failed, yellow for processing) for status indicators, ensuring quick visual recognition.
- A responsive, card-based dashboard layout optimized for clear presentation of statistics and tabular data. Key statistical metrics will be presented in prominent cards, followed by a list of recent entries in a clean table format. The primary navigation will reside in a persistent top bar.
- Subtle and swift animations for interactive elements like hover states on navigation items and cards, and smooth transitions when data is updated or loaded to enhance user experience without creating delays.
